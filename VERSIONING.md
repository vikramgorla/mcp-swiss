# Versioning & Release Process

See [`RELEASING.md`](./RELEASING.md) for the complete version strategy, branch model, and release workflow.

## Quick Reference

- **Semantic Versioning:** `MAJOR.MINOR.PATCH`
- **Minor** (`0.X.0`): new modules or tools
- **Patch** (`0.X.Y`): bug fixes, chores, docs
- **Branch model:** `develop` → feature branches → PRs. See [`CONTRIBUTING.md`](./CONTRIBUTING.md).
